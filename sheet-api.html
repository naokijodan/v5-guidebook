📊 PATTERN 03 ・ 一括登録シート V5

シート上でAPI翻訳
サイドパネルで進捗を見ながらバッチ翻訳 📊

スプレッドシート上でOpenAI APIを使って翻訳する方法です。サイドパネルで進捗を見ながら、200〜300行のまとまった件数も止まらずに翻訳できます😊

🙋

こんな人に

こんな方にぴったり
  • 外部ツールを入れずシート内で完結させたい
  • 自分でOpenAI APIを管理できる
  • 文字数やプロンプトの調整を自分でやれる
💰 コストは1品 約1〜2円（画像枚数で変動）。APIは本人が管理します。
🧠

この方法の仕組み

🔄 サイドパネルの「司令塔方式」で完走

シート上でOpenAI APIを呼んで翻訳します。サイドパネルが少しずつ（数行ずつ）処理を繰り返す「司令塔方式」なので、GASの6分制限にかからず、大量の行も最後まで完走します。

🎒

準備するもの

はじめる前に、これを用意します👇

🔑 OpenAI APIキー

platform.openai.com/api-keys で取得します。

OpenAIダッシュボードで Spending limits（利用上限）の設定を推奨します。想定外の課金を防げます。

⚙️

初期設定

最初に一度だけ。順番にすすめれば大丈夫です😊

1
API設定を開く

メニュー 🌐 eBay 翻訳 (AI) →　🔑 API設定 (APIキー登録) を開きます。

📌 開くダイアログの名称は ⚙️ Gemini 翻訳ツール 設定 ですが、OpenAIの設定もここで行います。名前に惑わされず進めてください。
📷 画像
2
APIキーとモデルを設定

基本は OpenAI でOKです。OpenAI 設定 の OpenAI API キー に sk- から始まるキーを貼り付け、OpenAI モデル は gpt-5.4-mini を選びます（とりあえず mini でOK）。

💡 Gemini のAPIキーを入れて試すこともできます。
📷 画像
3
担当者名（任意）

共通設定 の 担当者名 を入れると、v5インポートの担当欄に入ります。空のままでもOKです。

4
タグの登録（必須）

TagShipping シートのA列に、扱うジャンルのタグを必ず登録しておきます。このタグはカテゴリーIDの候補絞り込みにも使われます。

5
翻訳プロンプトの編集（任意）

メニュー 🌐 eBay 翻訳 (AI) →　🔑 翻訳プロンプト編集 から、専用シート 翻訳プロンプト の A2 を編集できます。空にするとデフォルトに戻ります。

🚀

基本の使い方

設定が終わったら、毎回の流れはこれだけ👇

1
作業範囲を選んでサイドパネルを開く

インポート用 シートで翻訳したい行を選び、メニュー 🌐 eBay 翻訳 (AI) →　🔑 サイドパネルでバッチ翻訳 を開きます。

📷 画像
2
行を確認・チャンクサイズを決めて開始

📥 選択範囲を取り込む で開始行・終了行を入れ、合っているか確認します。チャンクサイズ（1回に処理する行数） は 15 推奨（上限15）。あとは 開始 を押すだけです。進捗バー・成功／失敗の件数が表示されます。

📷 画像
3
途中でやめる・再開する

途中でやめたいときは キャンセル。ブラウザを閉じても、またメニューから開けば続きから再開できます。

4
完了を確認する

完了すると ✅ 処理完了！ と表示され、結果は v5インポート シートに追記されます（既存の行は上書きしません）。

5
カテゴリIDの空欄を最終調整

タグやカテゴリIDが出ていない行は、最終調整します。カテゴリIDが分からないときは、新しく作った ebayカテゴリー発見君 で日本語検索してIDをコピーし、カテゴリーID 列に貼り付けます。

🔎 ebayカテゴリー発見君を入れる →
📷 ①発見君（ストア）
📷 ②日本語検索→IDをコピー
6
仕入れ先コードをコピー

書込先 v5インポート に入ったデータの 仕入れ先コード の列だけをコピーします。

📷 画像
7
作業シートに「値のみ貼り付け」→ label を埋める

作業シートに 値のみ貼り付け します（そのまま貼ると条件付き書式が壊れるため）。そのあと左上の 翻訳orSKU ボタンを押すと label が生成されます。

📷 ①値のみ貼り付け
📷 ②翻訳orSKU
8
データをコピーして一括出品に貼り付け

v5出品用シートに生成されたデータを選択してコピーし、一括出品（EAGLE） に貼り付けて登録すれば完了です。

📷 ①データを選択してコピー
📷 ②EAGLEに貼り付け
🆔

カテゴリIDも自動で判定

🎯 翻訳と同時に最適なカテゴリーIDを選ぶ

翻訳と同時に、AIがeBay公式カテゴリーの候補から最適なIDを選び、システムが公式リストで照合してから記入します（許可タグをもとに候補を絞り込み）。

存在しない番号は記入されません。

⚠️

このパターンの手間・注意

✏️調整は自分で行う

文字数の微調整や、翻訳の癖を変えたいときのプロンプト編集は、自分で行う必要があります（精度はまずまず）。

🚫処理中はシートを触らない

処理中は対象のシートを手で編集しないでください。途中の書き込みとぶつかる恐れがあります。

🔁失敗行はやり直しボタンがない

失敗した行は件数で表示されます。失敗行だけをやり直すボタンはないので、その範囲を選び直して再度翻訳します。

外部AI（CLI・Codexアプリ）で翻訳
サブスクの範囲で、AIに任せる ✨

Codex・Claude Code・Gemini CLI などのAI自身に、スプレッドシートを直接読み書きさせて翻訳する方法です。OpenAIなどのAPI課金は不要で、お使いのAIサブスクの範囲内で動きます😊

🙋

こんな人に

こんな方にぴったり
  • すでにCLIやCodex／Claude等のサブスクを使っている
  • 追加のAPI課金を気にせず使いたい
  • 出力をAIと相談しながら仕上げたい・スキルを編集したい
👑 導入の壁と時間はかかりますが、いちばん融通が利く方法です。サブスクの範囲で動くので、追加のAPI課金もありません。
🧠

この方法の仕組み

🔄 AI自身の力でシートを読み書き

翻訳を外部APIに課金して呼ぶのではなく、いま使っているAI（Codex／Claude／Gemini）自身の力で、シートの日本語商品データ（テキスト＋画像）を読み、英語のeBay出品データに変換して書き戻します。

だから別途APIキーの従量課金は不要で、サブスク内で完結します。

🎒

準備するもの

はじめる前に、3つを用意します👇

🛠️ 使えるAIツール

次のいずれかがあればOKです。

  • Codex app
  • Codex CLI
  • Claude Code
  • Gemini CLI

「Codex アプリ」の導入がまだの方へ。インストール手順は別ガイドにまとめています👇

MCP google-sheets が、そのAI環境に接続済みであること。シートの読み書きに使います。

⚙️

初期設定

最初に一度だけ。順番にすすめれば大丈夫です😊

1
スキル本文を入手

シートのメニュー 🌐 eBay 翻訳 (AI) →　📄 スキル本文をダウンロード を開き、📥 ファイルとしてダウンロード または 📋 クリップボードにコピー を押します。

📌 本文中の ===（ここから下をコピーしてスキルに登録）=== より下を登録に使います。上は人向けの変更履歴なので、登録は不要です。
📷 ①メニューを開く
📷 ②DL／コピー
2
AIにスキルとして登録

お使いのAIに合わせて、本文を登録します。

登録のしかた
  • Codex app：Skills 設定で新規Skillを作成し、本文を貼り付け
  • Claude Code：~/.claude/skills/ebay-translation/SKILL.md に保存
  • Gemini CLI：skillフォルダに配置（対応）

コピーした本文を AIに渡すと、スキルを作ってくれます（Codex app なら貼り付けて実行するだけ）。

📷 画像
3
翻訳設定（任意）

メニュー ⚙️ 設定 で、こまかい動きを保存できます。基本は規定値のままでOKです。

設定できる項目
  • タグ参照シート既定 TagShipping
  • 書込先シート既定 v5インポート
  • ソースシート既定 インポート用
  • Skill名既定 ebay-translation
  • 担当者名（書込先の担当欄に入ります）
🚀

基本の使い方

設定が終わったら、毎回の流れはこれだけ👇

1
行を選んで翻訳指示文を作成

ソースシート インポート用 で翻訳したい行を選び、メニュー 🌐 eBay 翻訳 (AI) →　📋 選択行の翻訳指示文を作成 を開きます。

📷 画像
2
指示文を生成してコピー

開始行・終了行を確認して 生成 → 📋 クリップボードにコピー を押します。

📷 画像
3
AIに貼り付けて実行

コピーした指示文を Codex ／ Claude ／ Gemini に貼り付けて実行します。あとは結果を待つだけです。

📷 画像
4
最初の認証はスキップを選ぶ

最初はいくつか認証を求められますが、「今後尋ねない（スキップ）」を選んでおけば、あとは自動で最後まで実行してくれます。

⏳ かなり丁寧に処理するため、シートのAPI翻訳よりも時間はかかります。AIのモデルやプラットフォームによって癖があるので、スキルを調整しながら使うのがおすすめです。
📷 画像
5
画像もしっかり判定（Codexの場合）

商品画像も取り込んで判定してくれるので、テキストだけでは分からない情報も反映されます。

📷 画像
6
気になる点は会話で修正を依頼

出力に間違いや不満があれば、そのまま伝えると直してくれます（例：「タイトルをもう少し長く」「タグがフィギュアなのにアニメグッズになっている所を精査して」）。これがこの方法のいいところです😊

📷 画像
7
書き込まれる → 仕入れ先コードをコピー

結果が書込先 v5インポート に書き戻されます。入った商品の 仕入れ先コード の列だけをコピーします。

📷 ①書き込み結果
📷 ②仕入れ先コードをコピー
8
作業シートに「値のみ貼り付け」→ label を埋める

作業シートに 値のみ貼り付け します（そのまま貼ると条件付き書式が壊れるため）。そのあと左上の 翻訳orSKU ボタンを押すと label が生成されます。

📷 画像
9
データをコピーして一括出品に貼り付け

v5出品用シートに反映されたデータを選択してコピーし、一括出品（EAGLE） に貼り付けて登録すれば完了です。

📷 ①データを選択してコピー
📷 ②EAGLEに貼り付け

この方法ならではの強み

「一番おすすめ」と呼ぶ理由はここにあります👑

💸

コストを気にせず
たくさん翻訳

サブスクの範囲内なら、コストを気にせずたくさん翻訳できます。API課金の従量料金がかかりません。
💬

AIと会話して
仕上げられる

出力された訳を、AIと普通に会話して直してもらえます。たとえば「タイトルをもっと短く」のように頼めます。
✏️

ルール自体を
書き換えられる

スキル本文（ルール）自体を自分で書き換えて、翻訳の癖を調整できます。自分好みの仕上がりに育てられます。
🖼️

画像の扱い（メルカリ非経由）

🛡️安全な画像だけをAIに渡す

画像は、シートの専用URL（doGet）経由でGoogleにホストされた安全な画像だけをAIに渡します。翻訳のためにメルカリへ自動アクセスしません。

⚠️

注意

🧱導入に少しハードル

CLIやアプリの導入に少しハードルがあります（最初の環境準備）。慣れるまでは時間がかかるかもしれません。

⏳件数が多いと時間がかかる

1件ずつAIが処理するため、件数が多いと時間がかかります。大量にこなしたいときは、その分の待ち時間を見込んでおきましょう。
